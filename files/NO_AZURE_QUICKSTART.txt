==========================================

You can't access Azure right now?

Here's your 15-minute setup path:


OPTION A: SQLITE (Fastest, Zero Setup)
======================================

Best for: Starting immediately, learning, local development

Step 1: Install dependency (1 minute)
npm install better-sqlite3

Step 2: Create backend/src/storage.ts
Copy entire code from SQLITE_STORAGE_GUIDE.txt

Step 3: Update backend/src/index.ts
Add this at top:
import SQLiteStorageService from './storage';

Replace in server startup:
const storage = new SQLiteStorageService();
await storage.connect();

Step 4: Update backend/.env
Add one line:
DATABASE_PATH=./sentiment_analyzer.db

Step 5: Start backend
npm run dev

DONE! Database auto-creates as file: sentiment_analyzer.db

Cost: $0
Setup time: 5 minutes
Cloud migration: Later (export/import)

STEP-BY-STEP MONGODB ATLAS SETUP
================================

1. Go to: mongodb.com/atlas

2. Click "Start free"

3. Register:
   - Email: your@email.com
   - Password: your password
   - Organization Name: Your Project Name
   - Click "Create organization"

4. Create Project:
   - Project Name: "Sentiment Analyzer"
   - Click "Create Project"

5. Create Cluster:
   - Choose "Free M0" (512MB, free forever)
   - Region: Choose closest to you
   - Cluster Name: "sentiment-cluster"
   - Click "Create Cluster" (takes 5 min)

6. Create Database User:
   - Go to Database Access
   - Add Database User
   - Username: your_username
   - Password: your_password
   - Click "Add User"

7. Get Connection String:
   - Go to Clusters
   - Click "Connect"
   - Choose "Drivers" → "Node.js" → "3.12 or later"
   - Copy connection string
   - Replace <password> with your password
   - Add /sentiment_analyzer at end

   Should look like:
   mongodb+srv://your_username:your_password@sentiment-cluster.mongodb.net/sentiment_analyzer?retryWrites=true

8. Add to .env:
   MONGODB_URI=mongodb+srv://your_username:your_password@sentiment-cluster.mongodb.net/sentiment_analyzer?retryWrites=true

9. Copy storage.ts from MONGODB_STORAGE_GUIDE.txt

10. Update backend/src/index.ts with:
    import MongoDBStorageService from './storage';
    const storage = new MongoDBStorageService();
    await storage.connect();

11. npm run dev

12. Test:
    curl http://localhost:3000/api/coins

DONE! You're using cloud MongoDB! 🎉

TROUBLESHOOTING
===============

"Connection refused"
- Check connection string in .env
- Verify MongoDB service is running (if local)
- Whitelist your IP (if Atlas)

"Authentication failed"
- Check username and password
- Ensure @ is not in password (use URL encoding %40)
- Verify database user was created

"No collections found"
- Collections created automatically on first write
- Make sure to call API endpoint that saves data

"Connection string not working"
- Verify you copied entire string correctly
- Check for spaces or special characters
- Replace <password> and <cluster> placeholders
