SIDE-BY-SIDE CODE COMPARISON
=============================

All three use identical code structure:

interface StorageService {
  async saveCoinSnapshot(coin)
  async getCoinSnapshots(symbol, days)
  async getTopCoins(limit)
  async saveSentimentAnalysis(sentiment)
  async getSentimentAnalysis(symbol, date)
  async savePriceHistory(symbol, ohlcv)
  async getPriceHistory(symbol, days)
}

Only difference: Connection and initialization!

SQLite:
const storage = new SQLiteStorageService('./sentiment_analyzer.db');

MongoDB:
const storage = new MongoDBStorageService(process.env.MONGODB_URI);

Cosmos DB:
const storage = new MongoDBStorageService(process.env.MONGODB_URI);
// Same code! Just different connection string

SETUP INSTRUCTIONS BY CHOICE
============================

CHOICE 1: SQLite (EASIEST)
1. npm install better-sqlite3
2. Copy storage.ts code from SQLITE_STORAGE_GUIDE.txt
3. Add to .env: DATABASE_PATH=./sentiment_analyzer.db
4. npm run dev
5. Done! Database creates automatically

CHOICE 2: MongoDB Atlas (RECOMMENDED)
1. Go to mongodb.com/atlas
2. Create free account
3. Create free cluster (takes 5 min)
4. Create database user
5. Get connection string
6. npm install mongodb
7. Copy storage.ts code from MONGODB_STORAGE_GUIDE.txt
8. Add to .env: MONGODB_URI=mongodb+srv://user:pass@cluster...
9. npm run dev
10. Done!

CHOICE 3: Local MongoDB (If you prefer local)
1. Install MongoDB Community Edition
2. Start mongod service
3. npm install mongodb
4. Copy storage.ts code from MONGODB_STORAGE_GUIDE.txt
5. Add to .env: MONGODB_URI=mongodb://localhost:27017/sentiment_analyzer
6. npm run dev
7. Done!

CHOICE 4: Docker MongoDB (Universal)
1. Install Docker
2. Run: docker run -d -p 27017:27017 --name mongo mongo
3. npm install mongodb
4. Copy storage.ts code from MONGODB_STORAGE_GUIDE.txt
5. Add to .env: MONGODB_URI=mongodb://localhost:27017/sentiment_analyzer
6. npm run dev
7. Done!
