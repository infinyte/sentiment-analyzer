QUICK START GUIDE
================

STEP 1: LOCAL SETUP
-------------------

1. Create project directory:
   mkdir sentiment-analyzer
   cd sentiment-analyzer

2. Create subdirectories:
   mkdir -p backend/src frontend/src .github/workflows

3. Copy backend files:
   - Copy 02_BACKEND.ts → backend/src/index.ts
   - Copy backend-package.json → backend/package.json
   - Create backend/tsconfig.json (see below)
   - Create backend/.env (see below)

4. Copy frontend files:
   - Copy 03_FRONTEND.tsx → frontend/src/App.tsx
   - Copy frontend-package.json → frontend/package.json
   - Create frontend/tsconfig.json (see below)
   - Create frontend/vite.config.ts (see below)
   - Create frontend/index.html (see below)

STEP 5: AZURE DEPLOYMENT
------------------------

Prerequisites:
- Azure account (free tier)
- GitHub repository
- NewsAPI key
- Claude API key

Create Azure resources:

1. Create resource group:
   az group create --name sentiment-rg --location eastus

2. Create storage account:
   az storage account create \
     --name sentiment$(date +%s) \
     --resource-group sentiment-rg \
     --location eastus \
     --sku Standard_LRS

3. Create app service plan:
   az appservice plan create \
     --name sentiment-plan \
     --resource-group sentiment-rg \
     --sku FREE --is-linux

4. Create backend web app:
   az webapp create \
     --resource-group sentiment-rg \
     --plan sentiment-plan \
     --name sentiment-api-app \
     --runtime "node|18-lts"

5. Create frontend web app:
   az webapp create \
     --resource-group sentiment-rg \
     --plan sentiment-plan \
     --name sentiment-dashboard-app \
     --runtime "node|18-lts"

6. Set environment variables:
   az webapp config appsettings set \
     --resource-group sentiment-rg \
     --name sentiment-api-app \
     --settings \
       NODE_ENV=production \
       PORT=8080 \
       NEWSAPI_API_KEY=your_key \
       CLAUDE_API_KEY=your_key \
       ALLOWED_ORIGINS=https://sentiment-dashboard-app.azurewebsites.net

7. Configure CI/CD:
   - Get publish profiles from Azure
   - Add to GitHub secrets
   - Create .github/workflows/deploy.yml (see below)

File: .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Build Backend
        working-directory: ./backend
        run: |
          npm install
          npm run build
      
      - name: Build Frontend
        working-directory: ./frontend
        run: |
          npm install
          npm run build
      
      - name: Deploy Backend
        uses: azure/webapps-deploy@v2
        with:
          app-name: sentiment-api-app
          publish-profile: ${{ secrets.AZURE_BACKEND_PUBLISH }}
          package: ./backend
      
      - name: Deploy Frontend
        uses: azure/webapps-deploy@v2
        with:
          app-name: sentiment-dashboard-app
          publish-profile: ${{ secrets.AZURE_FRONTEND_PUBLISH }}
          package: ./frontend/dist

8. Push to GitHub and watch deployment

9. Verify:
   - Backend: https://sentiment-api-app.azurewebsites.net/api/health
   - Frontend: https://sentiment-dashboard-app.azurewebsites.net

TROUBLESHOOTING
---------------

Backend won't start:
- Check PORT environment variable
- Check API keys in .env
- Run: npm run build
- Check console for error messages

Frontend shows blank page:
- Check browser console (F12)
- Verify CORS settings on backend
- Check that API_URL is correct
- Clear browser cache

API returns 404:
- Verify backend is running
- Check that routes are registered
- Test health endpoint first

CoinGecko API slow:
- Normal rate limit is 10-50 calls/minute
- Add delay between requests
- Use in-memory cache
